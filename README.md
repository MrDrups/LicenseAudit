# LicenseAudit — Кабинет лицензий ПО

Информационная система для учёта и управления лицензиями программного обеспечения. Обеспечивает полный цикл работы с реестром лицензий: от создания и назначения компаниям до продления, отзыва и экспорта данных. Реализована ролевая модель доступа (RBAC), журналирование операций и заявочный процесс согласования.

## Ролевая модель (RBAC)

Система реализует три роли с набором разрешений, назначаемых через связку «роль → разрешения»:

| Функционал | ADMIN | LEAD | USER |
|---|:---:|:---:|:---:|
| Блок «Лицензии» | Просмотр | CRUD, отзыв, продление | Просмотр, создание заявки |
| Блок «Компании» | Просмотр | CRUD | Просмотр |
| Блок «Лицензионные планы» | Просмотр | CRUD | Просмотр |
| Блок «Администрирование» | Полный доступ | Нет доступа | Нет доступа |
| Управление ролями/разрешениями | Полный доступ | Нет доступа | Нет доступа |
| REST API (GET) | ✅ | ✅ | ✅ |
| REST API (POST/PUT/DELETE) | ✅ | ✅ | ❌ |
| Заявки: создание, редактирование, отмена | ✅ | ✅ | ✅ |
| Заявки: одобрение, отклонение | ❌ | ✅ | ❌ |

### Разрешения (permissions)

`LICENSES_VIEW`, `LICENSES_EDIT`, `COMPANIES_VIEW`, `COMPANIES_EDIT`, `PLANS_VIEW`, `PLANS_EDIT`, `ADMIN_EDIT`, `ROLES_EDIT`, `REQUEST_CREATE`

## Основной функционал

- **CRUD лицензий** — создание, редактирование, удаление записей о лицензиях с привязкой к компании и лицензионному плану
- **Поиск** — фильтрация по ключевым параметрам на стороне сервера (`@Query`)
- **Продление лицензий** — обновление даты окончания с фиксацией в журнале
- **Отзыв лицензий** — установка статуса «отозвана» без удаления из реестра
- **Лицензионные планы** — индивидуальные тарифные планы (название, макс. число пользователей, цена)
- **Управление компаниями** — ведение реестра организаций-клиентов
- **Заявочный процесс** — USER создаёт заявку → LEAD одобряет/отклоняет → лицензия появляется в реестре
- **Журналирование** — каждое действие с лицензией фиксируется (тип операции, автор, время, старое/новое значение)
- **Администрирование** — управление пользователями, ролями, разрешениями, просмотр логов и статистики
- **Экспорт** — выгрузка данных в Excel (`.xlsx`) и CSV
- **REST API** — полный набор CRUD-эндпоинтов для лицензий, компаний и планов
- **Swagger UI** — интерактивная документация API

## Быстрый старт

### Предварительные требования

- **Java 21** (JDK)
- **Docker** + **Docker Compose** (для контейнерного запуска)
- **PostgreSQL 15** (для локального запуска без Docker)

### Запуск через Docker Compose

```bash
# Сборка JAR
./gradlew bootJar

# Запуск приложения и базы данных
docker compose up --build
```

Приложение будет доступно: **http://localhost:8081**

### Локальный запуск (без Docker)

1. Запустите PostgreSQL на `localhost:5432` (база `postgres`, пользователь `postgres`, пароль `postgres`)
2. Выполните:

```bash
./gradlew bootRun
```

### Профиль `dev` (тестовые пользователи)

Для создания тестовых пользователей запустите приложение с профилем `dev`:

```bash
./gradlew bootRun --args='--spring.profiles.active=dev'
```

## REST API

Базовый путь: `/api`

### Лицензии

| Метод | Путь | Описание | Доступ |
|---|---|---|---|
| `GET` | `/api/licenses` | Список всех лицензий | Все авторизованные |
| `GET` | `/api/licenses/{id}` | Лицензия по ID | Все авторизованные |
| `POST` | `/api/licenses` | Создание лицензии | LEAD, ADMIN |
| `PUT` | `/api/licenses/{id}` | Обновление лицензии | LEAD, ADMIN |
| `DELETE` | `/api/licenses/{id}` | Удаление лицензии | LEAD, ADMIN |
| `POST` | `/api/licenses/{id}/revoke` | Отзыв лицензии | LEAD, ADMIN |
| `POST` | `/api/licenses/{id}/extend` | Продление лицензии | LEAD, ADMIN |

Аналогичные эндпоинты доступны для **компаний** (`/api/companies`) и **лицензионных планов** (`/api/license-plans`).

## Безопасность

- **Аутентификация** — форма логина Spring Security (`DaoAuthenticationProvider`)
- **Хеширование паролей** — BCrypt (адаптивная хеш-функция с уникальной солью)
- **Авторизация** — декларативные правила доступа к URL по ролям
- **CSRF** — включён для веб-форм, отключён для `/api/**`
- **Журналирование** — все операции над лицензиями записываются в `LL01_LICENSE_LOG`
- **Принцип минимальных привилегий** — каждая роль имеет только необходимые права
